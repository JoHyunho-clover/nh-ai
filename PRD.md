# PRD: AI 주식 분석 어시스턴트 (AI Stock Intelligence Platform)

**작성일**: 2026-06-24  
**상태**: MVP 기획 완료  
**팀 규모**: 1명 (개발자)

---

## Problem Statement

초보 개인투자자는 주식 투자 결정을 내릴 때 다음과 같은 문제를 직면하고 있습니다:

1. **정보 과부하**: 기술적 분석, 펀더멘털, 뉴스 등 여러 정보 출처를 종합하기 어려움
2. **전문성 부족**: 차트 패턴, 거시경제 신호 해석 능력 부족
3. **의사결정 근거 부재**: "왜 매수해야 하는가"에 대한 명확한 이유를 알 수 없음
4. **추천의 신뢰성**: 일반 투자 조언이 개인의 포트폴리오와 얼마나 맞는지 불명확
5. **시간 제약**: 50개 종목을 매일 모니터링할 시간이 없음

---

## Solution

**AI 주식 분석 어시스턴트**는 다음과 같이 문제를 해결합니다:

1. **11개 분석 레이어 통합**: Market Regime, Smart Money, 기술적 지표, 펀더멘털, 뉴스 감성, 거시경제를 자동 분석
2. **Explainable AI**: 모든 추천에 명확한 근거를 함께 제시
3. **신뢰도 점수**: 각 분석 결과의 신뢰도를 명시 (60% 이하는 경고)
4. **자기개선 루프**: AI가 추천 성과를 추적하고 2일마다 가중치를 자동 조정해 시간이 지날수록 정확도 향상
5. **자동화**: 매일 16:30 시장 마감 후 자동으로 모든 종목 분석 및 추천 제시
6. **AI 포트폴리오**: 추천 종목으로 모의 포트폴리오를 자동 구성하고 수익률 추적

## User Stories

### As a 초보 개인투자자

1. I want to 50개의 관심 종목에 대해 매일 매수/매도 추천을 받고 싶어서, so that 일일이 차트를 확인하지 않고도 투자 기회를 놓치지 않을 수 있다.

2. I want to 각 추천에 대해 "왜 이 종목을 사야 하는가"에 대한 명확한 근거를 알고 싶어서, so that 근거 없이 투자하는 실수를 피할 수 있다.

3. I want to 매수가/매도가 목표를 단계별로 받고 싶어서 (1차/2차/3차), so that 나의 자산 규모와 리스크 허용도에 맞춰 진입할 수 있다.

4. I want to 단기(1~3개월)와 장기(6개월~1년) 두 관점의 추천을 동시에 받고 싶어서, so that 단기 변동성과 장기 추세 모두를 고려한 투자 결정을 할 수 있다.

5. I want to 각 추천의 신뢰도를 표시해줬으면 좋겠어서, so that 신뢰할 수 있는 추천과 그렇지 않은 추천을 구분할 수 있다.

6. I want to AI가 얼마나 정확한지 확인하고 싶어서, so that 이 서비스를 믿고 사용할 수 있다.

7. I want to 내 포트폴리오의 AI 추천 종목 수익률을 자동으로 추적받고 싶어서, so that AI의 추천이 실제로 얼마나 수익을 냈는지 알 수 있다.

### As the AI System

8. I want to 추천 성과(매수가 도달? 목표가 달성? 손절가 터치?)를 자동으로 추적하고 싶어서, so that 내 가중치를 2일마다 자동 조정해 정확도를 높일 수 있다.

9. I want to 성공한 추천과 실패한 추천에서 각 분석 레이어의 기여도를 분석하고 싶어서, so that 정확한 레이어는 가중치를 증가시키고 틀린 레이어는 감소시킬 수 있다.

10. I want to 급격한 가중치 변동을 방지하고 싶어서 (+/- 2% 이내), so that 한두 번의 실패로 인한 과도한 조정을 피할 수 있다.

### As a Product Owner

11. I want to MVP로 최소 기능만 빠르게 완성하고 싶어서, so that 시장 검증을 빨리 할 수 있다.

12. I want to 실제 매매 기능은 제외하고 싶어서, so that 법적 책임과 복잡도를 최소화할 수 있다.

13. I want to 3개 종목(삼성전자, SK하이닉스, 현대차)으로 시연 가능하게 하고 싶어서, so that MVP 완성도를 쉽게 검증할 수 있다.

14. I want to 정보 보안과 개인정보를 최소화하고 싶어서, so that MVP 단계에서 회원가입 시스템을 제외할 수 있다.

## Implementation Decisions

### 1. 분석 엔진 아키텍처 (11 Layer Stack)

**결정**: 11개 분석 레이어를 별도 모듈로 설계하되, 각 레이어는 독립적으로 점수(0~100)를 생성합니다.

**이유**: 
- 각 레이어의 기여도를 독립적으로 추적 가능 (자기개선 루프용)
- 레이어별 가중치 조정이 수월함
- 향후 특정 레이어만 개선/교체 가능

**세부사항**:
```
Layer 1: Market Regime (15%)
Layer 2: Smart Money (15%)
Layer 3: Market Structure (15%)
Layer 4: Volume Profile (15%)
Layer 5: Technical Indicators (10%)
Layer 6: Candlestick Analysis (10%)
Layer 7: Chart Pattern Recognition (5%)
Layer 8: Sector Rotation (5%)
Layer 9: Fundamental Analysis (5%)
Layer 10: News Sentiment (3%)
Layer 11: Macro & Global (2%)
```

각 레이어는 OHLCV 데이터를 입력받아 JSON 형식의 점수와 근거를 반환합니다:
```json
{
  "layer_name": "Market Regime",
  "score": 85,
  "signal": "BUY",
  "confidence": 0.92,
  "rationale": "강세장 신호 + 외국인 순매수"
}
```

---

### 2. 자기개선 루프 (Self-Improving Engine)

**결정**: 추천 성과를 자동 추적하고, 2일마다 레이어별 가중치를 동적으로 조정합니다.

**피드백 루프**:
1. AI가 "삼성전자, 매수가 80,000원 / 매도가 88,000원" 추천
2. 각 레이어의 점수 스냅샷을 DB에 저장
3. 30일 후: 최저가가 80,000원에 도달했는가? → 매수가 도달 여부 기록
4. 90일 후: 최고가가 88,000원에 도달했는가? → 매도가 달성 여부 기록
5. 결과 분류: 성공(매수+매도) / 손절(매수+손절가) / 미도달(매수 미도달)
6. 레이어별 기여도 분석:
   - 성공한 추천: 정확한 레이어 가중치 +1% (최대 +2%)
   - 실패한 추천: 틀린 레이어 가중치 -1% (최대 -2%)
7. 다음 추천에 개선된 가중치 적용

**조정 빈도**: 2일마다 (지속적, 초기/중기/후기 구분 없음)

**안정화 장치**:
- ±2% 제한으로 급격한 변동 방지
- 누적 데이터 충분할 때만 조정 (초기 권장사항: 최소 10개 추천 후)

---

### 3. 데이터 아키텍처

**결정**: PostgreSQL + Redis 조합으로 OHLCV 데이터를 계층적으로 저장합니다.

**저장 전략**:
- **일봉(1D)**: 최근 5년 (거래일 1250개) → 기술적 분석용
- **주봉(1W)**: 최근 10년 → 중기 추세 분석용
- **월봉(1M)**: 최근 10년 → 장기 추세 분석용

**타임프레임 선택**:
- 단기 추천(1~3개월): 일봉 중심
- 장기 추천(6개월~1년): 주봉 + 월봉 중심
- 시장 구조 분석(Layer 3): 주봉 + 월봉

**갱신 주기**: Celery Beat로 매일 16:30 (시장 마감 후) 자동 갱신

---

### 4. AI 포트폴리오 (모의 투자)

**결정**: 백그라운드에서 AI 추천을 자동 실행하여 모의 포트폴리오를 관리합니다.

**사용자 입력**:
- 투자 가능 금액 (예: 1000만원)

**백그라운드 자동 실행**:
- AI가 매일 추천하는 종목으로 시뮬레이션 매수/매도 실행
- 실제 시장 데이터로 수익률 계산
- 추천 성과 자동 추적 (매수가 도달? → 매도가 달성? → 손절?)

**사용자 화면**:
- 누적 수익률 (%)
- 최근 2주 성과 분석
- 점수별 성공률

**중요**: 실제 거래 없음, 모의 투자만 수행

---

### 5. API 설계

**결정**: FastAPI로 RESTful 엔드포인트를 제공합니다.

**주요 엔드포인트**:
```
POST /watchlist
- 관심 종목 추가 (최대 50개)

GET /watchlist
- 전체 관심 종목 목록 + 최신 추천

GET /analysis/{symbol}
- 특정 종목의 상세 분석 리포트
- 응답: 단기/장기 추천, 점수, 신뢰도, 근거, 시나리오 확률

GET /portfolio
- AI 포트폴리오 누적 수익률

GET /performance
- AI 추천 정확도 통계 (최근 90일)
```

**응답 형식**: JSON
```json
{
  "symbol": "005930",
  "name": "삼성전자",
  "current_price": 82000,
  "short_term": {
    "signal": "BUY",
    "score": 88,
    "confidence": 0.92,
    "buy_prices": [80000, 78000, 75000],
    "sell_prices": [88000, 92000, 98000]
  },
  "long_term": {
    "signal": "BUY",
    "score": 76,
    "confidence": 0.85,
    "target_price": 105000
  },
  "scenarios": {
    "up": 0.62,
    "sideways": 0.25,
    "down": 0.13
  },
  "rationale": ["상승 추세 유지", "외국인 순매수 증가", ...]
}
```

### 테스트 모듈

**1. 데이터 수집 모듈 (Integration Test)**

**2. 각 Layer 모듈 (Unit Test + Integration Test)**
- Layer 1 (Market Regime): 강세장/약세장/횡보 판정이 맞는가?
- Layer 5 (Technical): MA, RSI, MACD 계산이 정확한가?
- 테스트: 과거 6개월 실제 데이터로 각 레이어 점수 생성 후 시각적 검증

**3. 가중치 통합 모듈 (Unit Test)**
- 11개 레이어 점수를 올바르게 가중치 적용했는가?
- 0~100점으로 정상 정규화되는가?
- 테스트: 고정된 레이어 점수 세트로 최종 점수 계산 검증

**4. 자기개선 루프 (Integration Test + 수동 검증)**
- 추천 기록이 정상 저장되는가?
- 30/90일 후 매수가/매도가 도달 여부를 정확히 판단하는가?
- 레이어별 기여도를 올바르게 분석했는가?
- 가중치 조정이 ±2% 범위 내인가?
- 테스트: 과거 1개월 실제 추천 데이터로 성과 자동 계산

**5. API 엔드포인트 (API Test)**
- `/analysis/{symbol}` 응답이 올바른 JSON 형식인가?
- 신뢰도 60% 이하 시 경고 필드가 포함되는가?
- 테스트: 수동으로 3개 종목에 대해 API 호출 확인

**6. UI 렌더링 (Manual Test)**
- 모바일 화면에서 레이아웃이 깨지지 않는가?
- 매수가/매도가가 3단계로 정상 표시되는가?
- 신뢰도 경고 아이콘이 보이는가?

1. **실제 자동 매매** — 주문 자동 실행 금지 (모의 투자만)

7. **유료 구독** — MVP 단계 무료
8. **모바일 앱** — 웹 기반만 (React 웹)
9. **다국어 지원** — 한국어만

# AI 주식 분석 어시스턴트 (AI Stock Intelligence Platform)

## 📋 프로젝트 개요

초보 개인투자자를 위한 AI 기반 투자 의사결정 지원 플랫폼

사용자가 등록한 관심종목(최대 50개)을 대상으로 AI가 기술적 분석, 수급 분석, 뉴스, 펀더멘털, 거시경제를 종합 분석하여 **매수/매도 추천**을 제공한다.

---

## 🎯 핵심 원칙

### 1. Explainable AI
모든 추천은 반드시 근거를 제공한다
```
매수 추천
근거
- 상승 추세 유지
- 외국인 순매수 증가
- 거래량 급증
- 반도체 업종 강세
- 긍정 뉴스 증가
```

### 2. Confidence Score
모든 분석 결과에 신뢰도를 표시한다
```
매수 의견
투자 점수: 84/100
신뢰도: 91%
```

### 3. 시나리오 기반 분석
단일 목표가 대신 확률 기반 시나리오 제시
```
상승 시나리오: 58% | 목표가 95,000원
횡보 시나리오: 27% | 가격범위 82,000~88,000원
하락 시나리오: 15% | 손절가 78,000원
```

### 4. 이중 관점 분석
**단기 관점(1~3개월)**: 기술적 분석, 차트 패턴 중심
**장기 관점(6개월~1년)**: 펀더멘털, 산업 전망 중심

### 5. 단기/장기 상충 신호 처리
단기와 장기 추천이 서로 반대일 때(예: 단기 매도 / 장기 매수),
AI가 가중치를 고려하여 **하나의 종합 의견**으로 정리한다.

```
단기 관점: 매도 (점수 62점)
장기 관점: 매수 (점수 81점)
AI 종합 의견: "보유 — 단기 조정 예상, 장기 상승 기조 유지"
```

### 6. 낮은 신뢰도 처리 정책
신뢰도 60% 이하일 경우 **숨기지 않고 그대로 표시**하되
⚠️ "신뢰도 낮음 — 참고용으로만 활용 권장" 경고를 함께 표시한다.
데이터 부족이 원인인 경우 사유도 함께 명시한다.

## 🏗️ 분석 엔진 구조 (11 Layer)

### Layer 1: Market Regime
현재 시장 상태 분석 (강세장/약세장/횡보장)
- 코스피, 코스닥, S&P500, VIX, 국채금리, 환율

### Layer 2: Smart Money Analysis
기관 및 외국인 자금 흐름 분석
- 외국인 순매수, 기관 순매수, 공매도, 대차잔고

### Layer 3: Market Structure
상승/하락/붕괴 추세 판단
- Higher High/Low vs Lower High/Low

### Layer 4: Volume Profile
거래량 기반 지지·저항 분석
- 강한 지지구간, 강한 저항구간

### Layer 5: Technical Indicators
MA20/60/120/240, RSI, MACD, Bollinger Bands, ATR, ADX, OBV, VWAP
- 단순 계산이 아니라 종합 해석

### Layer 6: Candlestick Analysis
최근 30~60개 캔들 패턴 분석
- Hammer, Morning Star, Bullish Engulfing, Shooting Star 등
- 캔들 + 거래량 + 지지선 + 시장상황 결합

### Layer 7: Chart Pattern Recognition
자동 패턴 탐지
- Cup and Handle, Head and Shoulders, Triangle, Wedge, Box Range 등

### Layer 8: Sector Rotation
업종 강도 분석
- 반도체, AI, 바이오, 방산, 전기차, 2차전지 등

### Layer 9: Fundamental Analysis
기업 가치 분석
- PER, PBR, ROE, EPS, 영업이익, 부채비율, 매출 성장률, FCF

### Layer 10: News Analysis
뉴스 및 공시 감성 분석
- 긍정/중립/부정 판정 및 감성 점수

### Layer 11: Macro & Global Analysis
국제 정세 분석
- 미국 금리, 연준 정책, CPI, 달러 인덱스, 원유, 지정학 리스크

```

   ↓
16:30~17:30 | 종가 데이터 수집 & AI 분석 엔진 실행

   ↓
09:00~09:30 | 사용자 앱 오픈 시 추천 결과 표시 (캐시 활용)
```

**제공 정보**
- 오늘 추천 종목 (매수/매도/보유)
- 단기(1~3개월) / 장기(6개월~1년) 이중 추천
- 투자 점수, 신뢰도, 근거

### 3. 포트폴리오 자동 분석
AI 추천 종목들로 자동 포트폴리오 구성
- 종목별 **AI 점수 기반 비중 배정** (점수 높을수록 비중 높게)
- 포트폴리오 성과 자동 추적
- 주간/월간/연간 수익률 분석
- 위험도(변동성, MDD) 분석
- AI가 자동으로 학습하고 포트폴리오 업데이트

**포트폴리오 비중 계산 예시**
```

- 현대차: 76점
- LG전자: 70점
- SK텔레콤: 65점

총합: 381점
비중: 88/381(23.1%) : 82/381(21.5%) : 76/381(19.9%) : 70/381(18.4%) : 65/381(17.1%)
```

## 📦 데이터 저장 원칙

### OHLCV 데이터
- 관심종목 전체에 대해 **최소 10년치 OHLCV 데이터** 저장
- 저장 단위: 일봉(1D), 주봉(1W), 월봉(1M)
- DB 저장 후 주기적(일 1회) 갱신

### 타임프레임 자동 선택
AI 분석 엔진은 분석 목적에 따라 타임프레임을 자동 선택한다:

| 분석 목적 | 사용 타임프레임 |
|-----------|----------------|
| 단기 관점 (1~3개월) | 일봉(1D) 위주 |
| 중기 관점 (3~6개월) | 일봉(1D) + 주봉(1W) |
| 장기 관점 (6개월~1년+) | 주봉(1W) + 월봉(1M) |
| 시장 구조 분석 (Layer 3) | 주봉(1W) + 월봉(1M) |
| 기술적 지표 (Layer 5) | 일봉(1D) |
| 차트 패턴 인식 (Layer 7) | 일봉(1D) + 주봉(1W) |

### 차트 이미지 원칙
- 차트 이미지는 **보조 수단**으로만 활용 (UI 시각화 목적)
- **분석의 기본 데이터는 원본 시세 데이터(OHLCV)**
- AI 엔진은 이미지가 아닌 수치 데이터를 직접 처리

## 🛠️ 기술 스택

### 백엔드
- **언어**: Python 3.11+

### 프론트엔드
- **프레임워크**: React 18+
- **모바일**: React Native 또는 Tailwind CSS (반응형)
- **상태관리**: Zustand 또는 Redux
- **차트**: TradingView Lightweight Charts / Chart.js

### AI & 데이터
- **AI**: Claude API (분석 및 인사이트 생성)
- **한국 주식 OHLCV**: yfinance (`005930.KS` 형식, 10년 데이터)
- **수급 데이터**: 네이버 금융 / 한경닷컴 웹 크롤링
  - 외국인 순매수, 기관 순매수, 공매도 잔고
- **뉴스 & 감성 분석**:
  - 네이버 금융 뉴스 크롤링
  - 다음 금융 뉴스 크롤링
  - NewsAPI (글로벌 영문 뉴스, 유료)
  - DART API (금융감독원 공시 — 무료)
- **거시경제 지표**: yfinance (`^KS11`, `^IXIC`, `^GSPC`, `^VIX` 등)
- **분석 라이브러리**: Pandas, NumPy, TA-Lib, scikit-learn

### 백테스트
- **엔진**: Backtrader 또는 VectorBT (과거 데이터로 전략 검증)

### 배포
- **컨테이너**: Docker
- **클라우드**: AWS (EC2, RDS, Lambda) 또는 GCP
- **CI/CD**: GitHub Actions

## 🔄 AI 자기개선 피드백 루프 (Self-Improving System)

이 시스템의 핵심은 AI가 자신의 추천을 스스로 검증하고 개선하는 것이다.
**실제 매매 실행은 사용자가 하며, 앱은 추천만 제공한다.**

### 피드백 루프 흐름

```
1️⃣ AI 추천 생성
   └─ "삼성전자, 매수가 80,000원 / 목표가 88,000원 / 손절가 74,000원"
   └─ AI 점수: 88점, 신뢰도: 92%

2️⃣ 추천 기록 저장 (DB)
   └─ 종목, 추천일, 매수가, 목표가, 손절가, AI점수, 신뢰도
   └─ 각 레이어별 점수 스냅샷 저장

3️⃣ 실제 시장 결과 자동 추적
   ├─ 30일 후: 최저가가 80,000원에 도달했는가?
   └─ 90일 후: 최고가가 88,000원에 도달했는가? 또는 손절가 74,000원 도달?

4️⃣ 결과 분류
   ├─ 성공 ✅: 매수가 도달 + 목표가 달성 (수익 실현)
   ├─ 손절 ⚠️: 매수가 도달 + 손절가 먼저 터치 (손실)
   └─ 미도달 ⏳: 매수가 자체 미도달 (진입 실패)

5️⃣ 레이어별 기여도 분석
   └─ 성공한 추천: 어떤 레이어가 정확했는가?
   └─ 실패한 추천: 어떤 레이어가 틀렸는가?

6️⃣ 가중치 자동 조정 (월 1회)
   ├─ 성공 예측 레이어 → 가중치 증가 (+2% 이내)
   ├─ 실패 예측 레이어 → 가중치 감소 (-2% 이내)
   └─ 급격한 변동 방지: 월 1회만 조정, ±2% 제한

7️⃣ 다음 추천에 반영
   └─ 개선된 가중치로 더 정확한 매수가/매도가 추천
```

### 추천 성과 추적 DB 스키마

```sql
-- 추천 이력
recommendation_history (
  id, symbol, created_at,
  
  -- 추천 데이터
  buy_price_1, buy_price_2, buy_price_3,    -- 1차/2차/3차 매수가
  target_price_1, target_price_2, target_price_3,  -- 1차/2차/3차 목표가
  stop_loss,  -- 손절가
  ai_score, confidence,  -- 0-100점, 0-100%
  
  -- 실제 시장 결과 (30일/90일 후 자동 채워짐)
  short_term_result,  -- success/stop/miss (30일 기준)
  long_term_result,   -- success/stop/miss (90일 기준)
  actual_low_30d, actual_high_90d,  -- 실제 최저/최고가
  
  -- 분석 정보
  layer_scores_json  -- {"layer1": 85, "layer2": 91, ...}
)

-- 레이어 가중치 변화 이력
layer_weight_history (
  id, updated_at,
  layer_name, old_weight, new_weight,  -- 예: 15% → 16.5%
  success_count, fail_count,  -- 조정 근거
  reason  -- 자동 생성 설명
)
```

### AI 성과 지표 대시보드 (사용자 화면)

```
📊 AI 추천 성과 요약 (최근 90일)

총 추천: 127건
├─ 매수가 도달: 89건 (70%)
│  ├─ 목표가 달성: 58건 (65%) ✅ 평균 +9.3% 수익
│  └─ 손절 발생: 15건 (17%) ⚠️ 평균 -4.1% 손실
│  └─ 보유 중: 16건 (18%) ⏳
└─ 미도달: 38건 (30%) (진입 기회 없음)

정확도 분석:
├─ AI 점수 80점 이상: 71% 성공률
├─ AI 점수 70~79점: 58% 성공률
└─ AI 점수 60~69점: 42% 성공률

월별 성과:
├─ 6월: 성공률 68%, 평균 수익 +8.2%
├─ 5월: 성공률 65%, 평균 수익 +7.9%
└─ 4월: 성공률 62%, 평균 수익 +6.8%
  └─ 추세: 점수 개선 중 ↗️
```

## 🔧 기술적 고려사항

### API 비용 관리
- Claude API: 일 50개 종목 × 1회 분석 = ~$0.50/일 (예상)
- 향후 월별 비용 모니터링 필요
- 무료 API로 우선 검증 후 필요시 유료화

### 에러 처리 전략
```
데이터 부족 (OHLCV < 10년)
 ↓
경고 메시지 + 사용 가능한 데이터로 분석
 ↓
신뢰도 자동 감소 (신뢰도 × 0.8)

API 실패 (한국투자증권 API)
 ↓
최근 캐시 데이터 사용
 ↓
"데이터 갱신 지연" 알림

Claude API 실패
 ↓
근거 자동 생성 (규칙 기반 텍스트)
 ↓
수동으로 분석 재실행
```

### 사용자 인증
- MVP: 회원가입 없음 (로컬 저장소 또는 세션 기반)
- 향후: OAuth 또는 JWT 추가

### 데이터 검증
- 일봉 데이터 250개 이상 (≈1년 거래일)
- 주봉 데이터 52개 이상 (≈1년)
- 월봉 데이터 120개 이상 (≈10년)
- 미충족 시 해당 타임프레임 분석 건너뛰기
